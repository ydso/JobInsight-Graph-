# 国家大学生就业服务平台招聘数据爬取方案

> 规划日期：2026-06-22  
> 当前状态：方案已完成，尚未实施  
> 适用范围：毕业设计中的公开招聘岗位采集与分析

## 1. 需求摘要

在使用者通过官网手工登录本人账号、不绕过验证码、不采集简历或联系方式的前提下，采集国家大学生就业服务平台（NCSS）账号可正常查看的职位列表与职位详情，输出与“前程无忧”项目兼容的中文表头 CSV、运行日志、质量汇总和分析数据集。

本方案复用“前程无忧”项目已经验证的数据工程思路：

- 采用相同的 10 个城市和 8 个数据类岗位，便于跨平台比较。
- 保留单城市单岗位采集、批量编排、SQLite 去重与断点续爬、质量汇总、Spark 多行 CSV 读取和单元测试。
- NCSS 使用独立来源指纹库，不复制“前程无忧”的 `fingerprints.db`；跨平台去重放到后续 Spark 清洗阶段。
- API 返回的专业标签只作为平台标签保存；只有岗位正文中出现明确专业要求证据，才写入“正文确认标准专业”。

明确不做：

- 不自动填写、读取或记录学信网/NCSS 账号凭据；登录必须由使用者在官网页面手工完成。
- 不访问个人中心、简历、投递记录、收藏、消息和企业管理后台。
- 不下载简历、附件或批量抓取企业联系人。
- 不破解验证码、不伪造登录态、不使用代理池规避限制。
- 不将原始岗位数据对外公开或用于商业用途。

## 3. 平台实测结论

以下内容于 2026-06-22 通过低频只读请求核验，实施前应再次执行烟雾测试。

| 项目 | 当前结论 |
|---|---|
| 官网 | `https://www.ncss.cn/` |
| 职位页 | `https://www.ncss.cn/student/jobs/index.html`，登录后可见结果可能更多 |
| 列表接口 | `GET /student/jobs/jobslist/ajax/`，返回 JSON |
| 地区代码接口 | `GET /student/code?type=areacodejc`，返回名称与代码 |
| 职位详情页 | `/student/jobs/{jobId}/detail.html`，复用同一浏览器登录会话 |
| 登录入口 | `/student/signin.html` 跳转学信网账号登录；程序仅打开官网页面并等待使用者手工完成 |
| robots 状态 | `/robots.txt` 当前返回普通 HTML，而非可解释的 robots 规则；按“未获得自动采集许可”谨慎处理 |

### 3.1 列表接口

请求示例：

```text
GET https://www.ncss.cn/student/jobs/jobslist/ajax/
    ?jobName=数据分析师
    &areaCode=110100
    &offset=1
    &limit=10
```

页面脚本当前使用的参数：

```text
jobType, areaCode, jobName, monthPay, industrySectors, property,
categoryCode, memberLevel, recruitType, offset, limit, keyUnits,
degreeCode, sourcesName, sourcesType
```

首版只使用 `jobName`、`areaCode`、`offset` 和 `limit`，避免扩大采集范围。`offset` 按页码处理；不能仅依赖响应中的总数，必须同时检查“本页是否为空”和“是否出现新岗位 ID”。

### 3.2 列表 JSON 已观察字段

```text
jobName, highMonthPay, lowMonthPay, updateDate, publishDate,
headCount, degreeName, recName, recLogo, areaCodeName, jobId,
recScale, recProperty, recTags, major, recruitType, sortPriority,
memberLevel, keyUnits, userType, recId, sourcesName,
sourcesNameCh, sourcesType
```

### 3.3 详情页已观察结构

- 岗位标题和公司名称：详情页头部。
- 薪资、学历、招聘人数、更新时间：`ul.salary` 区域。
- 实际地点和平台来源：岗位元信息区域。
- 岗位全文：`.mainContent-geshi`。
- 公司行业：`#mainindustries`。
- 涉及领域：`#industrySectors`。
- 公司性质、规模、网址、地址：公司详情列表，宜按中文标签取值，不按固定序号取值。

详情解析优先使用稳定 ID、class 和“标签名称 → 相邻值”；不得依赖整个页面的第 N 个 `li`。

### 4.3 数量与分析口径

- 每个岗位跨 10 城争取获得 500 条标题匹配、详情可解析的唯一记录。
- 城市×岗位不强求 500 条；公开结果耗尽即停止。
- 少于 30 条：`descriptive_only`，仅作描述性分析。
- 30–49 条：`limited`，分析时标注样本限制。
- 50 条及以上：`sufficient`。
- 公告型记录与具体岗位记录必须区分。`recruitType=1` 当前页面表现为“公告”，不能与普通岗位直接混算；默认保留但写入 `来源记录类型=announcement`，分析数据集默认排除。

## 5. 技术方案

### 5.1 总体策略

采用“公开 JSON 列表 + 公开 HTML 详情 + 单线程低频访问”的方式：

```text
任务生成
  → 请求公开列表 JSON
  → 标题相关性过滤
  → 来源指纹去重
  → 请求公开详情页
  → 详情解析与文本清洗
  → 内容指纹去重
  → 字段质量校验
  → 原子写入 CSV / 检查点 / 日志
  → 批次结束生成质量汇总和分析数据集
```

列表接口和详情页均使用工作区已有的 Playwright 浏览器上下文，不新增依赖。登录模式使用独立持久化浏览器资料目录，使列表接口请求与详情页共享同一会话。资料目录只保存在本机，不写入 CSV、JSON 或日志；若禁用登录，则使用临时浏览器上下文。

### 5.2 默认参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| 并发数 | 1 | 全局串行 |
| 列表页大小 | 10 | 与官网页面当前值一致 |
| 列表请求间隔 | 5–8 秒随机 | 每次请求完成后等待 |
| 详情请求间隔 | 6–10 秒随机 | 降低连续详情访问频率 |
| 单请求超时 | 30 秒 | 连接和读取总超时 |
| 单页网络/5xx 重试 | 3 次 | 20、40、60 秒退避 |
| 403/429 | 0 次立即重试 | 保存检查点并停止当前批次 |
| 最大页数 | 100 | 安全上限，通常会先因结果耗尽停止 |
| 连续空页停止 | 1 页 | API 空页即结束当前组合 |
| 连续无新 ID 停止 | 2 页 | 防止分页参数失效导致重复第一页 |
| 每日请求上限 | 1500 | 到上限保存检查点并退出 |
| 日志级别 | INFO | 不记录 Cookie、令牌或完整响应正文 |

### 5.3 HTTP 行为

- 使用固定、真实且可识别的普通浏览器 User-Agent，不做高频 UA 轮换。
- `Referer` 设为公开职位列表页。
- 每个组合开始前先请求列表第一页，不预先遍历全站。
- 请求间隔由单一全局节流器控制，重试也必须经过节流器。
- 只对网络超时和 5xx 做有限重试。
- 遇到 403、429、验证码、登录失效或异常 HTML，保存检查点并停止，不尝试规避；单条详情的普通超时只有限重试并跳过该条。
- 不将 CSRF 值、Cookie 或登录页内容写入日志和数据集。


NCSS 增加：

```text
招聘人数、更新日期、平台专业标签、来源记录类型、
平台来源名称、平台来源类型、精选企业标记、重点领域标记
```

字段规则：

- `source = ncss`。
- `schema_version = ncss-csv-v1`，字段变化必须升级版本。
- 规范详情链接固定为 `https://www.ncss.cn/student/jobs/{jobId}/detail.html`。
- 来源指纹：`SHA256("ncss|" + jobId)`。
- 内容指纹：`SHA256(标准公司名 + 标准岗位名 + 标准城市 + 清洗后描述)`。
- 平台专业标签不能单独生成“正文确认标准专业”；专业和技能证据规则与前程无忧一致。
- 详情页无法访问时，不用列表摘要伪造详情字段；记录失败原因并跳过分析数据集。

## 8. 实施步骤

### 阶段 A：锁定契约

1. 在 `test_crawler_ncss.py` 中先写列表 JSON、详情 HTML、时间戳、薪资、专业证据和 CSV 表头测试。
2. 从官网响应中截取最小必要测试夹具，删除 CSRF、Cookie、跟踪参数和无关正文。
3. 固定 `ncss-csv-v1` 中文表头，并与 `../前程无忧/batch_crawler.py:185-219` 的核心字段做集合一致性测试。

### 阶段 B：实现单次采集器

1. 实现地区代码、列表 URL 和详情 URL 构造。
2. 实现全局节流、超时、有限重试和登录跳转检测。
3. 解析列表 JSON，先做标题过滤和来源指纹去重。
4. 解析公开详情页，按稳定选择器和标签提取全文与公司字段。
5. 复用前程无忧的薪资、学历、经验、职责/要求、技能和专业判定行为。
6. 原子写入完整 CSV 与运行日志；默认不保存全量原始 JSON/HTML。

### 阶段 C：实现批量编排

1. 配置 10 城市、8 岗位和标题白名单。
2. 建立 NCSS 独立 SQLite 表：`fingerprints`、`content_fingerprints`、`crawl_progress`。
3. 每成功处理一页后更新 `next_page`；中断后从下一页继续。
4. 对结果耗尽、连续重复、达到日限额、403/429、登录跳转分别记录明确停止代码。
5. 批次结束只生成一个质量汇总和一个分析数据集。

### 阶段 D：数据质量与 Spark 接入

1. 生成城市×岗位和岗位总体两级质量记录。
2. 分析数据集默认排除公告、详情失败、标题不相关、疑似培训广告和重复记录。
3. 增加城市等权 `analysis_weight`，不改变岗位的真实城市归属。
4. 使用 `multiLine=true`、`header=true`、`escape='"'` 读取包含多行描述的 CSV。
5. 在 Spark 层计算跨源指纹，对 NCSS 与前程无忧的同公司、同岗位、同城市记录去重。

## 9. 数据质量规则

| 检查项 | 规则 | 处理 |
|---|---|---|
| 来源岗位 ID 为空 | `jobId` 为空 | 丢弃并记录 `missing_job_id` |
| 岗位名称/公司为空 | 清洗后长度小于 2 | 丢弃 |
| 标题不相关 | 不满足岗位白名单或命中排除词 | 丢弃并单独计数 |
| 分页重复 | 连续两页无新 `jobId` | 终止组合，标记 `pagination_repeated` |
| 详情登录跳转 | 最终 URL 为学信网/登录页 | 保存检查点并停止批次，要求使用者重新手工登录后续爬 |
| 详情正文过短 | 清洗后少于 20 个字符 | 不进入分析数据集 |
| 公告记录 | 映射为 `announcement` | 原始 CSV 保留，分析数据集默认排除 |
| 薪资均为 0 | 平台值为 0/0 | 写“面议”，数值列留空 |
| 招聘人数为 0 | 平台值为 0 | 标记“未说明”，不解释为零人 |
| API 专业标签无正文证据 | 只有 `major` 字段 | 只保留平台标签，不确认专业要求 |
| 技能处于否定语境 | 如“无需 Python” | 不计入技能候选 |
| 城市不匹配 | 实际地点与搜索城市不同 | 保留实际地点并加质量标记 |
| 疑似培训广告 | 命中培训、学费、包就业等组合规则 | 排除分析数据集 |

## 10. 合规与安全边界

- 仅访问使用者本人账号在官网正常可查看的职位列表、地区代码和职位详情。
- robots 规则当前无法从 `/robots.txt` 得到明确授权，因此不把“未禁止”解释为“允许批量抓取”。
- 上线批量采集前，应检查平台官网声明、用户协议和课题所在学校的数据使用要求；如果条款禁止自动化采集，应改用公开数据集、人工导出或书面授权。
- 原始数据仅用于本地毕业设计研究；论文或演示只展示汇总统计和脱敏示例。
- Cookie 仅由 Playwright 保存在本机专用浏览器资料目录；CSV、JSON 和日志不保存 Cookie、CSRF、账号、IP、简历、联系方式和投递信息，资料目录不得上传或共享。
- 不下载详情页附件；如研究确需附件，只记录公开附件存在性，并另行评估授权和版权。
- 一旦出现验证码、403、429 或登录要求，保存检查点并停止，不增加并发、不换代理、不绕过控制。

## 11. 验收标准

实施完成必须同时满足：

1. `python -m unittest test_crawler_ncss.py` 全部通过，退出码为 0。
2. 离线测试不访问互联网；网络烟雾测试必须通过显式参数启用。
3. 单次烟雾测试限定 1 城市、1 关键词、最多 2 条详情；登录模式只验证会话复用，不自动操作登录表单。
4. 烟雾测试的请求并发始终为 1，任意相邻请求间隔不少于配置下限。
5. 列表解析样本中，`来源岗位ID`、`岗位名称`、`公司名称`、`规范来源链接` 完整率为 100%。
6. 对 20 条公开详情抽样，正文解析成功率不低于 95%；失败项必须有停止/质量代码。
7. 同一次和重启后的输出均不存在重复 `来源岗位指纹`。
8. 两页连续返回同一批 ID 时，采集器在第二次无新 ID 后停止，不无限翻页。
9. 模拟 403、429、登录跳转时，不重试绕过，检查点能够恢复。
10. CSV 使用 UTF-8 BOM、中文表头，岗位描述中的换行不会拆成多条记录。
11. Spark 读取后的记录数与 CSV 逻辑记录数一致。
12. 平台专业标签在无正文证据时不会进入“正文确认标准专业”。
13. `爬取质量汇总_*.csv` 同时包含城市×岗位和岗位总体记录、停止原因、样本等级及去重统计。
14. `岗位分析数据_*.csv` 默认不包含公告、详情失败、标题不相关和疑似培训广告记录。

## 13. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 接口或 HTML 结构变化 | 解析失败 | 离线夹具测试 + 启动烟雾测试 + 选择器集中定义 |
| API 分页元数据不可靠 | 漏采或死循环 | 以空页、新 ID 数和连续重复为主，页数上限兜底 |
| 公告与岗位混合 | 样本失真 | 保存来源记录类型，分析集默认排除公告 |
| 专业标签被误当正文要求 | 专业统计偏高 | 平台标签与正文证据分列，沿用前程无忧证据规则 |
| 详情页触发访问限制 | 批次中断 | 单线程、低频、日上限、403/429 立即停止并续跑 |
| 城市样本量差异大 | 城市比较失真 | 样本等级 + 城市等权权重 + 报告中披露样本量 |
| 跨平台重复岗位 | 总量虚高 | 来源内 SQLite 去重，Spark 层再做跨源指纹去重 |
| 原始描述包含个人信息 | 数据合规风险 | 字段白名单、联系方式脱敏、原始数据不对外发布 |
| 词典快照发生分叉 | 跨源口径不一致 | 记录词典版本并测试两个项目版本一致性 |
