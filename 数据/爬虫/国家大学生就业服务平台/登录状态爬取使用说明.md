# NCSS 登录状态爬取使用说明

## 首次登录

在本目录执行：

```powershell
python ncss_crawler_standalone.py --login-only
```

脚本会打开 NCSS 官方登录页。请在浏览器窗口中手工完成账号、密码、短信或验证码操作；检测到“个人中心/我的简历/退出登录”等官网登录标志后，脚本会保存本机会话并退出。

登录资料默认保存在 `.ncss_browser_profile`。该目录包含敏感会话信息，不要上传、打包、共享或提交到版本库，也不要同时用两个爬虫进程打开它。

## 登录后采集

```powershell
python ncss_crawler_standalone.py --auth-mode required --city 北京 --job 数据分析师 --max-pages 3
```

`required` 会在采集前验证登录状态。会话失效时程序会直接停止并保留检查点，不会悄悄退化为未登录采集。重新运行 `--login-only` 后，再执行原采集命令即可续爬。

需要登录后立即采集，也可以执行：

```powershell
python ncss_crawler_standalone.py --login --auth-mode required --city 北京 --job 数据分析师 --max-pages 3
```

如果采集中途容易触发登录/验证码中断，建议开启人工恢复，并放慢请求节奏：

```powershell
python ncss_crawler_standalone.py --login --auth-mode required --manual-recovery --city 上海 --job 数据分析师 --interval 8 --detail-interval 2 --max-pages 50
```

开启 `--manual-recovery` 后，脚本会使用可见浏览器窗口运行；遇到登录失效或网站验证时，会打开官网页面等待你手工处理，检测恢复后重试当前列表页或详情页。它不会自动填写账号、密码或验证码，也不会绕过限流。若提示“访问过于频繁”或返回 429，脚本仍会保存检查点并停止，建议稍后再从检查点续跑。

注意：采集过程中不要关闭脚本打开的 Chromium 窗口；手动关闭窗口会同时关闭 Playwright 会话，后续接口请求会报 `TargetClosedError`，脚本只能保存检查点后停止。需要暂停时，先让当前命令结束，之后用同一命令续跑。

## 其他模式

- `--auth-mode auto`：默认模式；自动复用已有会话，没有有效会话时继续采集公开岗位。
- `--auth-mode required`：毕业设计正式采集建议使用；必须确认已登录，否则不开始。
- `--auth-mode off`：完全不读取持久会话，使用临时浏览器上下文。
- `--headful`：显示浏览器窗口，适合排查页面结构变化；常规采集可省略。
- `--manual-recovery` / `--pause-on-verification`：采集中遇到登录/网站验证中断时，等待手工处理后重试当前页。
- `--manual-recovery-timeout 秒数`：设置人工处理最长等待时间，默认 300 秒。
- `--profile-dir 路径`：指定其他专用登录资料目录。

## 稳定性边界

- 列表与详情共享同一个浏览器上下文，登录 Cookie 不需要手工复制。
- 未开启人工恢复时，401/403、登录跳转、验证码和 429 不会循环重试；程序保存检查点后停止。
- 开启人工恢复时，只对登录/验证码中断等待手工处理并重试；429/访问过于频繁仍会停止。
- 如果日志出现 `browser has been closed` 或 `browser_closed`，说明浏览器窗口/会话被关闭了，重新运行原命令即可从检查点续跑。
- 普通网络错误有限重试；单条详情连续失败时仅跳过该岗位，不中断整个城市和岗位组合。
- 采集保持单线程和请求间隔，不使用代理池、不自动处理验证码、不执行投递或收藏。
- 批量使用前应确认平台服务条款、学校课题要求和数据使用授权允许此用途。
