版本：V1.0  
日期：2026-06-18  
关联文档：[需求文档.md](./需求文档.md)、[技术设计文档.md](./技术设计文档.md)

## 1. 文档目的

本文档针对**前程无忧（51job）** 招聘平台，制定一套**技术可行、合规可控、风险可管理**的爬取方案。该方案作为项目数据采集模块的**可选补充通道**，仅在主数据源（重庆人才网、重庆市公共就业服务网、国家大学生就业服务平台等公共平台）数据量不足时启用。

**重要前提：本文档不鼓励突破平台反爬机制或违反用户协议。所有采集行为应仅限公开可访问的页面、低频访问、仅用于毕业设计学术研究。**

## 2. 平台分析

### 2.1 平台概况

| 项目 | 说明 |
|------|------|
| 平台名称 | 前程无忧 51job |
| 网址 | https://www.51job.com/ |
| 岗位类型 | 全职、实习、校招、兼职 |
| 数据特点 | 岗位信息丰富，字段结构较规整，校园招聘频道(nanjing.51job.com 等城市子站)与毕业生主题相关性较高 |
| 日均在线岗位量 | 百万级 |

### 2.2 目标数据字段

从 51job 岗位详情页可提取以下字段，与本项目岗位表字段对应：

| 51job 页面字段 | 本项目字段 | 提取可行性 |
|---------------|-----------|------------|
| 岗位名称 | job_title | ✅ 高，列表页和详情页均可见 |
| 公司名称 | company_name | ✅ 高 |
| 工作地点（城市+区） | city, district | ✅ 高 |
| 薪资范围（文本） | salary_text | ✅ 中，部分岗位显示"薪资面议" |
| 薪资范围（数值） | salary_min, salary_max | ⚠️ 需解析"8千-1.2万/月"等中文格式 |
| 学历要求 | education | ✅ 高 |
| 工作经验原文与标准文本 | experience_raw, experience | ✅ 高 |
| 工作经验上下限 | experience_min_years, experience_max_years | ✅ 高，保留“3-5年”等区间 |
| 工作经验要求类型 | experience_requirement_type | ✅ 高，区间/下限/精确/不限/应届 |
| 公司规模 | company_size | ✅ 中，在公司详情页 |
| 公司行业 | industry | ✅ 中 |
| 岗位类型 | job_type | ✅ 高（全职/实习/校招） |
| 发布时间 | publish_date | ✅ 高 |
| 岗位描述 | job_description | ✅ 高，详情页全文 |
| 任职要求 | requirement_text | ✅ 中，需从描述中分离 |
| 福利标签 | — | ✅ 高（五险一金、双休等标签） |

对外CSV统一使用中文表头。API返回的 `major1Str`、`major2Str` 属于隐藏检索标签，不能代表岗位正文中的专业要求；标准专业字段只依据岗位描述中带有学历/专业语境的明确证据生成。

### 2.3 页面结构特点

51job 的两级页面结构：

```
搜索列表页 (Search Result Page)
  → 岗位详情页 (Job Detail Page)
      → 公司信息页 (Company Page，部分字段需跳转)
```

- **搜索列表页 URL 模式**：`https://search.51job.com/list/{cityCode},000000,0000,00,9,99,{keyword},2,{page}.html`
- **详情页 URL 模式**：`https://jobs.51job.com/{city}/{jobId}.html`
- **城市子站模式**：`https://www.51job.com/{city}/`（如北京 beijing、上海 shanghai、重庆 chongqing）

## 3. 反爬机制分析

根据平台公开信息和社区经验，51job 主要采用以下反爬手段：

### 3.1 反爬措施清单

| 层级 | 反爬措施 | 触发条件 | 难度 |
|------|---------|---------|------|
| **网络层** | IP 频控 | 同一 IP 短时间大量请求 | ⭐⭐⭐ |
| **网络层** | 人机验证（滑块/图形验证码） | 高频或异常访问模式 | ⭐⭐⭐⭐ |
| **协议层** | User-Agent 校验 | 非浏览器 UA | ⭐⭐ |
| **协议层** | Referer 校验 | 直接访问详情页无来源 | ⭐⭐ |
| **协议层** | Cookie/Session 校验 | 无有效会话 | ⭐⭐⭐ |
| **应用层** | JS 动态渲染关键数据 | SPA 渲染、AJAX 加载 | ⭐⭐⭐⭐ |
| **应用层** | 关键字段加密/混淆 | 薪资等字段经 JS 解密 | ⭐⭐⭐⭐ |
| **应用层** | WebDriver 自动化检测 | 检测 Selenium/Playwright 特征 | ⭐⭐⭐⭐ |
| **业务层** | 搜索结果分页上限 | 搜索结果通常限制前 50-100 页 | ⭐⭐ |
| **业务层** | 登录态要求 | 部分数据需登录后访问 | ⭐⭐⭐ |

### 3.2 robots.txt 分析

51job 的 robots.txt（`https://www.51job.com/robots.txt`）通常对以下路径有限制：

- 用户简历、个人中心相关路径
- API 接口路径
- 后台管理路径

**岗位搜索和详情页通常不在 robots.txt 明确禁止列表中**，但用户协议对自动化访问有额外限制。

### 3.3 用户协议限制

前程无忧用户协议（`https://www.51job.com/bo/service.php`）中对复制、转载、抓取等行为有明确限制条款，主要包括：

- 禁止未经许可的自动化数据采集
- 禁止使用爬虫、蜘蛛等程序获取网站内容
- 禁止对网站内容进行复制、传播、转载用于商业用途

**合规底线：仅采集少量公开岗位信息用于学术研究，不批量爬取、不商业使用、不传播原始数据。**

## 4. 爬取策略设计

### 4.1 总体策略：小规模 + 低频 + 公开页面 + 学术用途

本方案采用 **"最小必要原则"**，仅在主数据源不足时补充使用：

| 参数 | 设置 |
|------|------|
| 采集目标 | 仅 10 个核心城市 × 8 个核心岗位；城市组合按相关结果耗尽停止，不强求等量 |
| 岗位总体目标 | 每个岗位跨10城争取达到500条标题匹配的有效唯一岗位 |
| 采集总量上限 | 10,000-15,000 条原始记录（约为首版总目标的 30%-40%） |
| 请求间隔 | 列表页 ≥ 10 秒/页，详情页 ≥ 5 秒/页 |
| 每日采集时间窗口 | 凌晨 1:00-6:00（低负载时段） |
| 并发数 | 1（单线程串行） |
| 每日请求总量上限 | ≤ 2000 次 |

### 4.2 采集流程

```text
┌─────────────────────────────────────────────────────┐
│ 1. 任务生成                                          │
│    读取采集配置 → 按城市+岗位关键词生成采集任务队列      │
│    示例: 北京-数据分析师, 上海-算法工程师, ...         │
├─────────────────────────────────────────────────────┤
│ 2. 搜索列表页采集                                     │
│    构造搜索 URL → 请求列表页 → 解析岗位卡片             │
│    提取: 岗位ID、名称、公司、城市、薪资文本、发布时间、   │
│          详情页URL                                     │
├─────────────────────────────────────────────────────┤
│ 3. 详情页采集                                        │
│    请求详情页 → 解析岗位描述 → 识别职责/要求段落        │
│    提取: 完整岗位描述、任职要求、公司信息、福利标签       │
├─────────────────────────────────────────────────────┤
│ 4. 去重检查                                          │
│    计算指纹: SHA256(岗位名+公司名+城市+描述前200字符)    │
│    与已有数据库比对，跳过重复岗位                       │
├─────────────────────────────────────────────────────┤
│ 5. 数据落盘                                          │
│    原始HTML保存到HDFS Bronze层                        │
│    解析后JSON保存到本地缓存，批量写入HDFS               │
│    采集日志写入PostgreSQL/MySQL                        │
└─────────────────────────────────────────────────────┘
```

### 4.4 反爬应对措施

以下措施按**合规性从高到低**排列，首版仅启用 Tier 1-2，Tier 3 仅在极端必要时启用且需评估合规风险：

#### Tier 1：基础合规措施（默认启用，无合规风险）

| 措施 | 实现方式 | 说明 |
|------|---------|------|
| 合理请求间隔 | `time.sleep(5-10)` | 模拟人工浏览节奏 |
| 真实 User-Agent | 使用主流浏览器 UA | 如 Chrome/Firefox 最新稳定版 UA |
| 完整请求头 | 设置 Referer、Accept、Accept-Language | 模拟浏览器正常请求 |
| Cookie 维持 | 使用 `requests.Session()` | 维持正常会话状态 |
| 时间窗口控制 | APScheduler 定时启停 | 仅凌晨低负载时段采集 |
| 增量去重 | SHA256 指纹比对 | 减少无效请求 |

#### Tier 2：必要增强措施（默认启用，低合规风险）

| 措施 | 实现方式 | 说明 |
|------|---------|------|
| UA 池轮换 | 维护 5-10 个主流浏览器 UA，随机选用 | 避免单一 UA 指纹 |
| 动态页面渲染 | Playwright（仅详情页必要时） | 处理 JS 渲染内容 |
| 请求失败重试 | 单页默认重试 3 次，退避：20s → 40s → 60s | 避免连续失败 |
| 代理 IP 备选 | 预留代理配置，暂不启用 | 仅 IP 被封后人工评估是否使用 |
| 断点续采 | 每页原子写入检查点，并同步 SQLite 页码、数量和状态 | 重启后先用官网分页控件定位到检查点页，再继续采集 |
| 待补采队列 | 认证或单页重试耗尽时入队，默认补采 2 次 | 保留检查点，不丢弃当前组合 |
| 相关性枯竭停止 | 至少读取5页后，连续5页没有新增有效岗位即结束组合 | 正常结束并进入下一个城市，不加入失败重试 |

#### Tier 3：高级措施（默认不启用，需评估合规风险）

| 措施 | 风险 | 启用条件 |
|------|------|---------|
| Playwright 接管浏览器 | 中：WebDriver 检测 | 仅详情页纯静态无法获取时 |
| 代理 IP 池 | 中-高：可能违反协议 | IP 被封且主源数据严重不足时 |
| Cookie 手动导入 | 低：仅复用浏览器会话 | 搜索页需登录时 |

**原则：宁可不采，不突破平台安全机制。**

### 5.3 核心代码框架

#### 5.3.1 Scrapy Spider 主逻辑

```python
# crawlers/spiders/job51_spider.py
import hashlib
import yaml
import time
import scrapy
from scrapy import Request
from datetime import datetime


class Job51Spider(scrapy.Spider):
    name = "51job"
    
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        with open("crawlers/config/51job_config.yaml", "r", encoding="utf-8") as f:
            self.config = yaml.safe_load(f)
        self.request_count = 0
        self.total_crawled = 0
        self.max_total = self.config["collection"]["max_total_records"]
    
    def start_requests(self):
        """生成城市 × 关键词的组合任务"""
        for city_key, city_info in self.config["cities"].items():
            for keyword in self.config["keywords"]:
                if self.total_crawled >= self.max_total:
                    self.logger.info(f"已达采集上限 {self.max_total}，停止生成任务")
                    return
                
                url = self._build_search_url(city_info["code"], keyword, page=1)
                yield Request(
                    url,
                    callback=self.parse_list,
                    meta={
                        "city_key": city_key,
                        "city_name": city_info["name"],
                        "keyword": keyword,
                        "page": 1,
                        "max_per_combo": self.config["collection"]["max_per_city_keyword"]
                    },
                    dont_filter=True
                )
    
    def _build_search_url(self, city_code, keyword, page=1):
        """构造搜索URL"""
        template = self.config["source"]["search_url_template"]
        return template.format(city_code=city_code, keyword=keyword, page=page)
    
    def parse_list(self, response):
        """解析搜索列表页，提取岗位卡片信息"""
        meta = response.meta
        
        # 解析列表页岗位卡片（具体选择器需根据实际页面结构调整）
        job_cards = response.css("div.joblist-item")  # 示例选择器
        
        for card in job_cards:
            if self.total_crawled >= self.max_total:
                return
            
            # 提取岗位基础信息
            job_id = card.css("::attr(data-jid)").get()
            job_title = card.css(".job-title::text").get()
            company_name = card.css(".company-name a::text").get()
            salary_text = card.css(".salary::text").get()
            detail_url = card.css(".job-title a::attr(href)").get()
            publish_date = card.css(".publish-date::text").get()
            
            if not detail_url:
                continue
            
            # 计算去重指纹
            raw_fingerprint = f"{job_title}|{company_name}|{meta['city_name']}"
            fingerprint = hashlib.sha256(raw_fingerprint.encode()).hexdigest()[:16]
            
            # 请求详情页，传递列表页已提取的字段
            detail_meta = {
                **meta,
                "job_id": job_id,
                "job_title": job_title,
                "company_name": company_name,
                "salary_text": salary_text,
                "publish_date": publish_date,
                "fingerprint": fingerprint,
                "source_url": detail_url
            }
            
            time.sleep(self.config["rate_limit"]["detail_page_interval_sec"])
            yield Request(detail_url, callback=self.parse_detail, meta=detail_meta)
        
        # 翻页逻辑
        if meta["page"] < 100:  # 搜索结果通常100页为上限
            time.sleep(self.config["rate_limit"]["list_page_interval_sec"])
            next_page = meta["page"] + 1
            next_url = self._build_search_url(
                response.meta.get("city_code", ""), 
                meta["keyword"], 
                next_page
            )
            yield Request(
                next_url,
                callback=self.parse_list,
                meta={**meta, "page": next_page}
            )
    
    def parse_detail(self, response):
        """解析岗位详情页"""
        meta = response.meta
        
        # 提取完整岗位描述
        job_description = response.css("div.bmsg.job_msg::text").getall()
        job_description = "\n".join([t.strip() for t in job_description if t.strip()])
        
        # 识别"任职要求"段落
        requirement_text = self._extract_requirement(job_description)
        
        # 提取公司信息
        company_industry = response.css("div.com_msg p:contains('行业')::text").get()
        company_size = response.css("div.com_msg p:contains('规模')::text").get()
        
        # 构建统一岗位Item
        item = {
            "source": "51job",
            "source_url": meta["source_url"],
            "source_job_id": meta.get("job_id", ""),
            "crawl_time": datetime.now().isoformat(),
            "job_title": meta.get("job_title", ""),
            "company_name": meta.get("company_name", ""),
            "city": meta["city_name"],
            "salary_text": meta.get("salary_text", ""),
            "education": self._parse_education(job_description),
            "experience": self._parse_experience(job_description),
            "job_type": "全职",
            "industry": company_industry or "",
            "company_size": company_size or "",
            "publish_date": meta.get("publish_date", ""),
            "job_description": job_description,
            "requirement_text": requirement_text,
            "fingerprint": meta["fingerprint"],
        }
        
        self.total_crawled += 1
        yield item
    
    def _extract_requirement(self, text):
        """从描述中提取任职要求段落"""
        import re
        patterns = [
            r'任职要求[：:]\s*(.*?)(?=岗位职责|福利|公司简介|$)',
            r'任职资格[：:]\s*(.*?)(?=岗位职责|福利|公司简介|$)',
            r'岗位要求[：:]\s*(.*?)(?=岗位职责|福利|公司简介|$)',
        ]
        for pattern in patterns:
            match = re.search(pattern, text, re.DOTALL)
            if match:
                return match.group(1).strip()
        return text  # 未找到明显分段，返回全文
    
    def _parse_education(self, text):
        """从文本中提取学历要求"""
        import re
        edu_map = {
            "博士": "博士", "硕士": "硕士", "研究生": "硕士",
            "本科": "本科", "大专": "大专", "中专": "中专",
        }
        for key, val in edu_map.items():
            if key in text:
                return val
        return ""
    
    def _parse_experience(self, text):
        """从文本中提取经验要求"""
        import re
        match = re.search(r'(\d+[-~]\d+年|应届|不限|经验不限)', text)
        return match.group(1) if match else ""
```

#### 5.3.3 中间件：频率限制

```python
# crawlers/middleware/rate_limit_middleware.py
import time
from datetime import datetime


class RateLimitMiddleware:
    """控制请求频率"""
    
    def __init__(self):
        self.last_request_time = 0
        self.daily_count = 0
        self.daily_reset_date = datetime.now().date()
        self.max_daily = 2000  # 与 config 保持一致
    
    def process_request(self, request, spider):
        # 每日上限检查
        today = datetime.now().date()
        if today != self.daily_reset_date:
            self.daily_count = 0
            self.daily_reset_date = today
        
        if self.daily_count >= self.max_daily:
            spider.logger.warning(f"已达每日上限 {self.max_daily}，暂停请求")
            raise Exception("Daily limit reached")
        
        # 请求间隔控制（区分列表页和详情页）
        is_list_page = "list" in request.url
        interval = 10 if is_list_page else 5
        elapsed = time.time() - self.last_request_time
        if elapsed < interval:
            time.sleep(interval - elapsed)
        
        self.last_request_time = time.time()
        self.daily_count += 1
```

# crawlers/pipelines/dedup_pipeline.py
class DedupPipeline:
    """基于指纹的去重Pipeline"""
    
    def __init__(self):
        self.seen_fingerprints = set()
    
    def process_item(self, item, spider):
        fp = item.get("fingerprint", "")
        if fp in self.seen_fingerprints:
            spider.logger.debug(f"跳过重复岗位: {item.get('job_title')} @ {item.get('company_name')}")
            raise scrapy.exceptions.DropItem(f"重复岗位: {fp}")
        self.seen_fingerprints.add(fp)
        return item

class HDFSPipeline:

## 6. 数据质量与去重策略

### 6.1 去重逻辑

采用**三级去重**机制：

| 级别 | 方法 | 作用域 |
|------|------|--------|
| 一级：来源指纹 | SHA256(平台\|岗位ID) | 同一组合及跨页去重 |
| 二级：内容指纹 | SHA256(公司名\|清洗后岗位描述) | 识别更换岗位ID的重复发布 |
| 三级：SQLite指纹库 | 持久化来源指纹和内容指纹 | 跨批次去重 |

### 6.2 数据字段校验

| 校验项 | 规则 | 处理 |
|--------|------|------|
| 岗位名称为空 | `len(job_title) < 2` | 丢弃 |
| 岗位标题不匹配 | 不符合目标岗位对应的标题白名单 | 丢弃并单独统计，不计为重复岗位 |
| 翻页方式 | 点击官网分页控件，不依赖地址栏的 `pageNum` | 防止前端忽略 URL 参数并重复返回第 1 页 |
| 分页响应错位 | API `pageNum` 与分页控件目标页不一致 | 不处理该响应，按单页失败策略重试 |
| 岗位描述过短 | `len(job_description) < 20` | 丢弃 |
| 技能处于否定语境 | “无需掌握Python”“Python非必需”等 | 不计入技能候选 |
| API隐藏专业标签 | 正文没有对应专业证据 | 不纳入正文专业候选、标准专业和专业类别 |
| 疑似培训广告 | 描述含"培训""包就业""学费"等 | 丢弃 |
| 薪资格式异常 | 无法解析为数值区间 | 保留原文，标记为未解析 |
| 城市不匹配 | 页面城市与搜索城市不一致 | 以页面为准 |

### 6.3 与其他数据源的去重

51job 采集的数据可能与重庆人才网、国家大学生就业服务平台等来源存在重复（同一企业同一岗位在多个平台发布），跨源去重采用：

```python
def cross_source_fingerprint(job):
    """跨数据源统一指纹"""
    import hashlib
    normalized_title = job["job_title"].strip().lower().replace(" ", "")
    normalized_company = job["company_name"].strip().lower().replace(" ", "")
    normalized_city = job["city"].strip()
    raw = f"{normalized_title}|{normalized_company}|{normalized_city}"
    return hashlib.sha256(raw.encode()).hexdigest()
```

此指纹在 Spark 清洗阶段统一计算，避免同一岗位因来源不同被重复计入需求统计。

### 6.4 城市样本与跨城市补量

城市与岗位构成独立分层，不再要求每个城市岗位都达到500条。数据较多的城市可以补足某岗位的总体样本，但不得修改城市归属，也不得用于冒充数据较少城市的样本。

| 城市×岗位有效数量 | 样本等级 | 使用建议 |
|------------------|---------|---------|
| 少于30条 | `descriptive_only` | 样本较少，仅作描述性分析 |
| 30-49条 | `limited` | 可以分析，但必须说明样本量限制 |
| 50条及以上 | `sufficient` | 适合一般城市特征分析 |

每次批量采集结束后只生成两个分析成果文件：

- `岗位分析数据_*.csv`：保留各城市实际采集到的全部有效岗位，同时增加 `analysis_weight`；同一岗位下每个城市的权重总和为1，可直接用于全量分析或城市等权分析。
- `爬取质量汇总_*.csv`：通过 `record_type` 区分城市×岗位记录和岗位总体记录，同时保存停止原因、样本等级、城市数量及跨城市500条目标完成情况。

原始岗位数量只作为采集审计指标，不再作为城市组合成功或失败的判定条件。

## 7. 安全与容错

### 7.1 异常处理策略

| 异常类型 | 处理方式 |
|----------|---------|
| 网络超时 | 单页重试 3 次，间隔递增（20s/40s/60s） |
| HTTP 403/429 | 暂停当前城市-关键词组合 30 分钟 |
| HTTP 5xx | 暂停采集 5 分钟，最多重试 3 次 |
| 验证码页面 | 保存检查点并加入待补采队列，不尝试自动识别或绕过 |
| 页面结构变化 | 记录日志，标记解析失败，人工检查 |
| IP 被封 | 停止当日采集，记录被封时间和 IP |
| HDFS 写入失败 | 先写本地文件缓存，重试 HDFS 写入 |

### 7.2 数据安全

| 措施 | 说明 |
|------|------|
| 不存储 Cookie/登录凭证 | 仅采集公开页面，不需要登录 |
| 不存储个人简历数据 | 即使页面中包含也主动过滤 |
| 原始数据仅存 HDFS | 不直接暴露到外网 |
| 采集日志脱敏 | 日志中不记录完整 URL（避免带 token） |

## 10. 方案总结

### 10.1 核心原则

| 原则 | 说明 |
|------|------|
| **最小必要** | 只在主源不足时启用，采集量控制在最低需要 |
| **合规优先** | 不绕过反爬机制，不违反协议，仅采公开页面 |
| **学术用途** | 仅限于毕业设计研究，不传播、不商用 |
| **可追溯** | 完整记录来源URL、采集时间和采集方式 |
| **渐进式** | 从小规模测试开始，逐步验证再扩大 |
